## 2. Learn the four-step integration loop

Every partner integration ships in the same shape. Read these short concept pages before you write code — then keep them open as reference.

<CardGroup cols={2}>
  <Card title="1. Authentication" icon="key" href="/guides/getting-started/authentication">
    Exchange a `serviceName` + `serviceSecret` for a Bearer JWT. Scopes on the token enforce which facilities, schemes, and endpoints you can use.
  </Card>
  <Card title="2. Environments" icon="globe" href="/guides/getting-started/environments">
    Four tiers: local mock, sandbox, staging, production. Know which base URL to call and which secret manager to pull from.
  </Card>
  <Card title="3. The scheme field" icon="code-branch" href="/guides/getting-started/scheme-field">
    One top-level field — `CBHI` or `RAMA` — routes every multi-scheme request to the right backend and the right business rules.
  </Card>
  <Card title="4. Alternate lookup keys" icon="key-skeleton" href="/guides/getting-started/lookup-keys">
    Search facilities by FOSA, TIN, matricule or name; practitioners by licence, NID or phone. Kwivuza resolves any key you already hold.
  </Card>
  <Card title="5. Errors & codes" icon="triangle-exclamation" href="/guides/getting-started/errors">
    Every error uses the same envelope `{ error: { code, message, details } }`. Branch on `code`, never parse messages.
  </Card>
  <Card title="6. Claim lifecycle" icon="timeline" href="/guides/getting-started/claim-lifecycle">
    Submitted → Under review → Approved or Rejected → Paid. Pull the timeline any time; resubmit after rejection with one call.
  </Card>
  <Card title="7. Webhooks" icon="bell" href="/guides/getting-started/webhooks">
    Register an HTTPS endpoint, receive HMAC-signed event callbacks. At-least-once delivery with exponential-backoff retry.
  </Card>
</CardGroup>
